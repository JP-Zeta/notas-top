1. Nombres Significativos
Usa nombres descriptivos para variables, funciones y clases (ej: calculateTotal() en lugar de calc()).

2. Funciones Cortas y Específicas
Cada función debe hacer una sola cosa y hacerla bien (idealmente < 20 líneas).

3. Comentarios Útiles (no redundantes)
Explica el "por qué" (no el "qué") del código. Evita comentarios obvios como:

javascript
// Incrementa i en 1  
i++;  

4. Evitar Código Duplicado (DRY)
Extrae lógica repetida en funciones reutilizables.

5. Estructura Consistente
Sigue un mismo estilo (indentación, llaves, etc.) en todo el proyecto.

6. Manejo de Errores Elegante
Usa try/catch para excepciones y evita fallos silenciosos.

7. Optimizar Legibilidad sobre Rendimiento (en la mayoría de casos)
El código debe ser fácil de entender primero; optimiza solo en cuellos de botella.

8. Pruebas Unitarias
Escribe tests para asegurar que cada parte funcione como esperas.

9. Refactorizar Constantemente
Mejora el código existente en lugar de parchearlo.

10. KISS (Keep It Simple, Stupid)
La simplicidad es clave. Evita sobreingeniería.


El código te dice cómo, los comentarios te dicen por qué funciona.
